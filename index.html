Concepts

There are two concepts: lections and folders.
A lection is a collection of vocabulary. If you have a lot
of vocabulary, lections help you focus on learning just a part
of it. You can also specify parts of a lection
using the join menu.
Learn more about that in 09Joining Lections/Lists Together.

With a folder you can organize your lections in different topics.
To create a folder, name your lection using the following scheme: "folder_name/lection_name"
In the lections view menu (07), folder names will be highlighted and
in the view menu, if you type as lection "folder_name/", it will show the content of all lections
in the specified folder name.

Next - 02Entering Vocabulary

Entering Vocabulary

Entering vocabulary can be done through the Add Menu.
Press "a" in the Main Menu to open the Add Vocabulary menu.
First, it will ask you for a lection. Lections are explained in
01Concepts. Press Enter to move to the Key field.
Here, you can enter the word that will be shown as a hint for the value.
For example, if you're learning a foreign language with TinyVoc,
you can enter the foreign word here.
After pressing Enter again, you can enter the value — in this case,
the word in your own language. This is the word you will need to type.

Next - 03Training Vocabulary

Training Vocabulary

Training vocabulary can be done through the Training Menu.
Press "t" in the main menu to open the Train Vocabulary menu.
First, it will ask you for a lection that you want to learn.
Lections are explained in 01Concepts. Press Enter
after entering the lection you want to learn. You’ll get a random key
that you set in the Add Vocabulary menu. Learn more about how to enter
vocabulary in 02Entering Vocbularuy.
You will see two values next to the key — values you didn’t set manually
in the Add Vocabulary menu. It looks like this: "Hello (4,2)".
The first number shows how many times you got the vocabulary wrong,
and the second shows how many times you got it right. These values
can also be seen in the View Menu. Learn more about the View Menu in
04Viewing Vocabulary.
When you enter the value, press Enter to verify it. If it's correct,
it will light up green and move to the next word — this word won’t be
asked again in the current training session. If it's incorrect, it will
light up red and be asked again. If you get it wrong three times in a row,
it will light up yellow and show the solution. Press Enter to go to the next key.
That vocabulary will be repeated later.
When entering correct answers, you’ll see a progress bar above the lection,
along with two numbers like this: "1/5". The first number shows how many
words you’ve answered correctly in the current lection; the second is the
total number of vocabulary items.
After finishing the training, you’ll see statistics including:
Total Correct, Total Wrong, Hardest Word, Average Attempts (average tries
needed to get a word right), and Success Score.
You can exit this by pressing "-".

Next – 04Viewing Vocabulary

Viewing Vocabulary

Viewing vocabulary can be done through the View Menu.
Press "v" in the main menu to open the View Vocabulary menu.
First, it will ask you for a lection you want to view.
You can also enter "all" to see the contents of all lections.
If you don't know which lections you have, read
07Viewing all Lections/Lists.
The table will show you the unique ID of each vocabulary item,
along with the Key, Value, Lection, and Wrong/Right counts
If you print this page using your browser’s print dialog,
all non-essential elements will be removed, so only the table is shown.

Next – 05Generating Lections/Lists

Generating Lections/Lists

One of the most powerful tools in TinyVoc is the
Generate List menu. It can transform lections/lists
using five different filters. Open this menu by pressing "g"
in the Main Menu. It will ask you for a list to transform.
Enter the lection you want to transform. Pressing Enter
will then ask you for a transform action. There are a total
of five transform actions, represented by numbers from zero to four.
Once all arguments/parameters are provided, a new lection
will be generated. The name of the new lection will be shown on screen.

Action Zero sorts the given list. It takes one argument —
the column by which to sort. For example, entering "wrong" will sort
the list so that vocabulary with the most wrong attempts appear at the bottom.
You can also enter columns like "key".
If the first character is a letter, sorting is alphabetical.
If it's a number, sorting will be numerical.

Action One reverses the given list.

Action Two checks each vocabulary entry and removes it
if a specified column is under a given value.
The first parameter is the maximum value allowed.
The second parameter specifies which column to check.

Action Three is similar to Action Two,
but removes entries if the specified column is above a given value.

Action Four shortens a lection to a specific length.
For example, if your lection has 5 entries and you specify "2"
as the first parameter, the output list will only contain the first two entries.

Next – 06Renaming Lections

Renaming Lections

Renaming vocabulary can be done through the Move Menu.
Press "m" in the main menu to open the Move Vocabulary menu.
First, it will ask you for the source lection you want to rename.
Pressing Enter will take you to the next step. In the destination field,
enter the new name for the lection.
You’re probably wondering why it's called "Move List" instead of "Rename List".
This is inspired by a trick in Linux, where the mv (move) command

Editing Already Entered Vocabulary

Editing already entered vocabulary can be done through the Correct Menu.
Press "c" in the main menu to open the Correct menu.
First, enter the lection that contains the vocabulary you want to correct.
Then, enter the ID of the vocabulary entry. You can find the ID in the View Menu.
Learn more about the View Menu in 04Viewing Vocabulary.
After pressing Enter, the Key and Value fields will be auto-filled.
You can now edit the content and press Enter again to proceed to the next field.

Next – 09Joining Lections/Lists together

Joining Lections/Lists Together

Joining multiple lections into one can be done through the Join Menu.
Press "j" in the main menu to open the Join menu.
You will be presented with a single text field called "Lections".
To specify which lections should be included in the final output lection,
enter the lections separated by commas. For example, if you enter
"lection1,lection2", the output lection will contain the vocabulary
from both "lection1" and "lection2".
More advanced usage is possible by adding brackets ("[]") after the lection name.
For example, entering "lection1[6]" will make the output lection contain
only the vocabulary with ID 6. You can find the vocabulary ID in the View Menu.
To learn more about the View Menu, read 04Viewing Vocabulary.
You can also specify ranges, such as "lection1[6-12]", which will include
the vocabulary with IDs between 6 and 12 (inclusive).
If you specify "lection1[4-]", it will include all vocabulary from lection1 starting from ID 4.
You can also use the reverse range: "lection1[-4]" will include the last 4 items from lection1.
To exit the Join Menu use C-r

Next – 10Sharing Lections/Lists with Other Users

Sharing Lections/Lists with Other Users

Sharing lections with other users can be done through the Broadcast Menu
and the Export menu entry. However, it's recommended to use the Broadcast Menu
instead of the Export option. Press "b" in the main menu to open the Broadcast Menu.
There are two different ways to share your lections: via file or WebRTC.
WebRTC is powered by PeerCloud and PeerJS. It uses an external server,
which may go down for a few hours or without notice.

Press "f" to use the file-sharing option. Here you can enter, just like in the Join Menu,
comma-separated lections — but bracket support (`[]`) is not available here.
To learn more about the Join Menu, read 09Joining Lections/Lists together.
The export will be immediately downloaded as a `voc.tiny` file.
You can later import it using the Import menu.
To learn more about the Import Menu, read 11Importing Vocabulary.

Use WebRTC to send your vocabulary over the network.
First, you need to set up a receiver — the device that should receive the data.
On the receiving device, go to the Broadcast Menu with "b", then press "w"
to open the WebRTC submenu, and then "r" to start receiving.
A 32-character peer ID will be shown on the screen.

On the sending device, also open the Broadcast Menu with "b", then "w"
to enter the WebRTC submenu, and press "s" to select the Send option.
Now you’ll need to enter the peer ID. Add the peer ID to the URL’s hash
(after the `#` symbol), but do not reload the page.
Pressing Enter in the browser's address bar should trigger the `hashchange` event.
If successful, it will confirm that the peer ID was saved and ask you for the lections.
Enter them comma-separated. Just like with file export, bracket notation (`[]`) is not supported here.
To learn more about the Join Menu, read 09Joining Lections/Lists together.

After the client receives the data, the lections will be automatically added.

Show the Learning History

TinyVoc stores a history of the attempts and success scores of all lessons.
To view this history, press "f" in the main menu to open the "Figure Menu".
You will see a lesson name, and below it a diagram with, on the left, the last
twenty average attempt scores, and on the right, the last twenty success scores
in percent.

Next – 12 Reading the Bar

Reading the Bar

At the bottom, you will always see a bar. On the bottom left, it shows
which menu you are currently in. To the left of the Menu Display, you will
see which Docpage contains a description for the current menu. If it says
"Error: No Docs", this means that this menu has no documentation.

Next – 13 Importing Lessons/Lists from a File

Importing Lections/Lists from a File

To import data from a voc.tiny file, press "i" in the main menu.
This will replace the entry with a file selector. Click on "browse" to open
a file selection dialog from your operating system. Select the voc.tiny file.
It doesn’t need to have the .tiny extension — these are just renamed JSON files.
After selecting the file, it will ask if you want to add or overwrite.
Press "b" to select add, or "o" to select overwrite.
Overwriting means all your current vocabulary will be deleted and replaced
with the contents of the file.
Choosing add will keep all existing vocabulary and add new entries from the file.

Next – 14Git

Git

The Git integration is for advanced users who already know Git.
I won’t explain what commits or branches are — you’ll need to learn
that on your own. This section explains where the Git features are
and how I implemented them for users familiar with Git.
To access the Git integration, press "n" in the main menu to open the Git submenu.
You can also monitor the Git status by pressing "s" in the main menu — similar to running git status.

Next – 15Creating a repo

Creating a repo

Just like in Git, you need to initialize it first.
Press "i" in the Git submenu to initialize it.
This will enable all Git features, and the
Git status menu will start showing information
about the repository.

Next – 16Adding/removing Lections/Lists from Tracked Lections/Lists

Adding/removing Lections/Lists from Tracked Lections/Lists

Like in Git, you need to add files to tracking.
In TinyVoc, you need to add Lections to tracking.
Press "a" in the Git submenu to add a Lection to tracking.
Press "r" in the Git submenu to remove a Lection from tracking.
Untracked Lections will also appear in the Git status menu.

Next – 17Committing vocabulary

Committing vocabulary

To commit your vocabulary, press "c" in the
Git submenu. It will ask you for a
commit message. You can verify the commit
with git log. To learn more about
git log, read 18Show Commits.

Next – 18Show Commits

Show Commits

In Git, you use git log to see all
commits. Here, press "l" in the Git submenu
to list all commits.

Next – 19Rollback to older commit

Rollback to older commit

To replace your current vocabulary with an older
commit, use the cherry-pick option.
Press "h" in the Git submenu to start.
It will ask you for a commit hash.
You can get the commit hash by pressing "l" in the
Git submenu. To learn more about showing commits,
read 18Show Commits.
After entering the hash, it will check
for differences between the cherry-picked
commit and your current state. If there’s a difference,
a conflict will be shown. The only way to resolve
a conflict in TinyVoc's Git integration is to force.
It will ask if you want to force the replacement
of your current data. If you say yes, it will replace
the current data with the selected commit,
but it won’t auto-commit it.

Next – 20Branches

Branches

Like Git, TinyVoc’s Git integration supports branches,
but they’re quite basic. You can only create,
switch, list, and delete branches. No merging.
When you initialize TinyVoc’s Git integration,
it will create a branch called "master"
and copy your current vocabulary to it.
The "master" branch cannot be deleted.

Next – 21Creating Branches

Creating Branches

To create a branch, open the Git submenu.
Press "b" to enter the branch submenu,
then press "c" to create a branch. It will ask
for a branch name. After entering it,
the branch will be created.

Next – 22Showing Branches

Showing Branches

To list all branches, open the Git submenu.
Press "b" to open the branch submenu,
then press "l" to list all branches.
The current branch is prefixed with "* ".

Next – 23Deleting Branches

Deleting Branches

To delete a branch, open the Git submenu.
Press "b" to open the branch submenu,
then press "d" to delete a branch.
It will ask for the name of the branch to delete.
You cannot delete your current branch
or the "master" branch.

Next – 24Switching between Branches

Switching between Branches

To switch branches, open the Git submenu.
Press "e" for checkout. It will ask which
branch you want to switch to.
You can verify the switch using "branch list".
To learn more, read 22Showing Branches.
Next - 25Configuring TinyVoc

Configuring TinyVoc

You can configure TinyVoc in the configuration menu by pressing "e"
in the main menu. You will find multiple options. You can select
an entry by pressing "j" and "k"; arrow keys are also allowed.
The currently selected item is marked with the "=>" prefix.
Press Enter to start editing; the prefix will change to ">".
After pressing Enter, it will be saved. Note: you cannot change
the type of an entry that is a boolean.
Be careful—some changes can break TinyVoc. If TinyVoc breaks,
add "#ResetConfig" at the end of the URL you are using TinyVoc with.
This will reset TinyVoc's configuration to the default. Next - 26Troubleshooting

Troubleshooting

If you made a mistake configuring TinyVoc, read 25 Configuring TinyVoc.
There you will find help to reset your configuration. If some functions don’t work, or if the "TinyVoc" text
does not appear at the top, you might get help from one of the contributors. TinyVoc is an
open-source project on GitHub. You can create issues on GitHub via the Issues tab at the following link:
https://github.com/ls-root/tinyvoc/issues
Click "New Issue" to create one. Please provide details. To get them, right-click on the page,
click "Inspect", and then go to the "Console" tab. Copy any red error messages into the description.
Also, add the TinyVoc version. To see it, add #version at the end of the URL.
